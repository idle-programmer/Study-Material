
*⁠What is Index/indexing?
->Index is a specialized data structure (usually B-Tree) created on db columns to speed up data retrieval 
by allowing the database to locate rows faster instead of scanning the entire table,
and indexing is the technique of using these structures to improve query performance on db tables.
Advantages: Faster SELECT queries, Faster WHERE filtering, Faster JOIN/ORDER BY
Disadvantages: Slower INSERT / UPDATE / DELETE , Takes extra disk space

* What is Compound Index?
->A compound index (or composite index) is a database index that references multiple fields within a 
collection or table, allowing for faster query filtering and sorting across those fields.
syntax:
CREATE INDEX idx_orders_customer_date ON orders (customer_id, order_date);


*⁠What is transaction in SQL?
->A SQL transaction groups one or more db operations (like INSERT, UPDATE, DELETE)
into a single atomic unit that either fully succeeds (COMMIT) or fully fails (ROLLBACK).
syntax:
START TRANSACTION; -- or BEGIN 
UPDATE accounts SET balance = balance - 100 WHERE id = 1; 
UPDATE accounts SET balance = balance + 100 WHERE id = 2; 
COMMIT; -- Save changes 
-- ROLLBACK; -- Undo if error


*Normalization
->Normalization reduces redundancy and improves data integrity by organizing data into proper tables.
1NF: Atomic values, no repeating groups.
2NF: 1NF + no partial dependencies (non-prime attributes depend on full key).
3NF: 2NF + no transitive dependencies.
Higher forms like BCNF handle edge cases.

*⁠ACID properties -> Set of rules that ensure reliable and consistent transactions in a database.
Atomicity: A transaction is all or nothing
Consistency: Transaction moves the database from one valid state to another
Isolation: Multiple transactions run independently
Durability: Once a transaction is committed, data is permanent

* What is Transaction, Commit and Rollback in sql?
-> 
A transaction is a group of one or more SQL operations executed as a single unit of 
work to ensure data consistency and integrity.

COMMIT: Permanently save the changes made during a transaction.
ROLLBACK: Undo all changes if an error occurs, made during the current transaction.
